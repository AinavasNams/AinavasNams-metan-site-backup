## 🔧 **Codex CLI / GitHub Integration Task Specification**

> 📝 **Как использовать этот документ**
>
> 1. Открой новый чат в Codex CLI.
> 2. Вставь всё содержимое этого файла целиком как первый промпт.
> 3. Сразу после вставки добавь отдельной строкой: `Codex CLI включён, можно начинать аудит по этому ТЗ.`
>
> После этого можно начинать работу по плану ниже.

### 🎯 Цель

Запустить **аудит и ремонт кода сайта `metan.lv`**, используя подключение к репозиторию
`AinavasNams/AinavasNams-metan-site-backup`, ветка `backup-prod-2025-10-29`.

Я (AI-агент) должен:

1. Читать файлы напрямую из GitHub через активный Codex Connector.
2. Анализировать реальный код, а не вставки из чата.
3. Создавать ветки, патчи (в формате `*** Begin Patch ... End Patch`) и Pull Requests.
4. Подготавливать проект к корректному билду и прод-деплою.

---

### 📂 Файлы для аудита

1. `components/SimpleServiceCalculator.tsx`
   → проверить UX, адресное поле, автодистанцию, сабмит формы, отсутствие чекбокса согласия, вызовы аналитики.

2. `components/AddressAutocomplete.tsx`
   → проверить UX автодополнения, закрытие списка, видимость input, передачу координат.

3. `app/kontakti/page.tsx`
   → проверить корректность формы обратной связи, наличие чекбокса согласия, логирование ошибок, поведение после отправки.

4. `app/api/send-email/route.ts`
   → проверить корректность обработки SMTP и JSON-данных, логирование, чтение переменных окружения.

5. `app/layout.tsx`
   → проверить GTM-интеграцию, подключение аналитики, корректность SSR и `<noscript>` блока.

6. `lib/trackEvent.ts`
   → проверить, как реализован `trackEvent`, безопасен ли он для SSR и можно ли его унифицировать для всех компонентов.

### ⚙️ Планы и PR-ветки

1. **`pr/calculator-ux-distance+consent`**

   * исправление `SimpleServiceCalculator.tsx` и `AddressAutocomplete.tsx`;
   * добавление чекбокса согласия;
   * исправление логики авторасстояния;
   * замена импортов аналитики на локальные stub-функции.

2. **`pr/contacts-form-consent+success`**

   * улучшение UX формы контактов;
   * добавление чекбокса согласия и success-блока после отправки;
   * безопасный трекинг событий в `dataLayer`.

3. **`pr/smtp-stabilize+env`**

   * рефакторинг `app/api/send-email/route.ts`;
   * чтение SMTP-переменных из `.env.production`;
   * унификация структуры JSON-заявок.

4. **`pr/layout-gtm-env+cleanup`**

   * вынести GTM-ID в `NEXT_PUBLIC_GTM_ID`;
   * почистить дубли клиентских компонентов;
   * добавить `<meta>` и `<manifest>` при необходимости.

### 🧩 Алгоритм действий Codex-CLI

1. Подключиться к GitHub (репозиторий: `AinavasNams/AinavasNams-metan-site-backup`).
2. Проверить наличие ветки `backup-prod-2025-10-29`.
3. Считать указанные файлы.
4. Сгенерировать подробный отчёт об ошибках и несоответствиях.
5. Для каждой зоны сгенерировать отдельный патч.
6. Создать соответствующие ветки и PR.
7. Подготовить лог изменений (diff summary).

---

### 🧠 Результат
